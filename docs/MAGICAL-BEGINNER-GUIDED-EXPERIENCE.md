# Magical Beginner Guided Experience

## Welcome

This guide is for people who are new to computers, new to NVDA, or both.
You do not need technical knowledge.
You only need curiosity and a few minutes at a time.

By the end of this guide, you will know how to:

1. Save useful text quickly.
2. Reuse saved text without retyping.
3. Stay oriented so you do not feel lost.
4. Organize notes, tasks, and reminders.
5. Get help when something is not clear.

## How to Use This Guide

1. Move in order from top to bottom.
2. Try each short activity once.
3. If a step feels confusing, skip it and continue.
4. Come back later. Progress still counts.

## Your First Wins in 10 Minutes

### Step 1: Open the command palette

Use this when you are not sure what to do next.

1. Press NVDA+Shift+P.
2. Listen for the palette opening.
3. Type one word like "help" or "note".
4. Press Enter on the action you want.

What success feels like:

You hear a clear confirmation and the action starts right away.

### Step 2: Save something important quickly

Use quick capture for thoughts, links, and short reminders.

1. Select text in any app, or copy text to clipboard.
2. Press Grave+Q.
3. Hear confirmation that capture was saved.

What success feels like:

You can stop worrying about losing the idea.

### Step 3: Ask "what can I press"

When you forget keys, ask the system directly.

1. Press Grave+Slash.
2. Listen to available hotkeys for your current context.

What success feels like:

You get unstuck in seconds.

## Daily Flow for Beginners

### Morning setup

1. Open your task list.
2. Check diary entries for today.
3. Review your top note category.
4. Start one mission and finish one small action.

### During work or study

1. Capture ideas quickly with Grave+Q.
2. Use summaries for long text when needed.
3. Save reusable phrases to text expansion.
4. Keep related notes linked together.

### End of day

1. Mark completed tasks.
2. Add one short diary entry.
3. Export or back up if you made major changes.

## Feature Tour by Real-Life Goal

### Goal: "I want to remember this"

Use:

1. Quick Capture Inbox.
2. Notes workspace categories.
3. Attachments and note links.
4. Snapshot restore points.

Beginner tip:

Create one category named "Today" and one named "Later". Keep it simple.

### Goal: "I need to reuse text fast"

Use:

1. Clip slots to copy and paste saved snippets.
2. Clip browser to review and compare saved snippets.
3. Text expansion for short abbreviations that expand into full text.

Beginner tip:

Start with only three clip slots. Add more later if needed.

### Goal: "I want cleaner writing"

Use:

1. Rewrite for beginners.
2. Markdown helpers.
3. Semantic HTML generation.
4. Accessibility lint and fix preview.

Beginner tip:

If a result sounds too complex, run rewrite again and ask for simpler language.

### Goal: "I need to stay organized"

Use:

1. Tasks and completion tracking.
2. Diary entries by date.
3. Date and time helpers.
4. Optional calendar sync.

Beginner tip:

Write short tasks like "Call doctor" instead of long paragraphs.

### Goal: "I need to find information"

Use:

1. Retrieval query.
2. Result revisit memory.
3. Retrieval summary.
4. Action extraction.

Beginner tip:

If results are noisy, ask one specific question instead of a broad one.

### Goal: "I want safe recovery"

Use:

1. Operation journal.
2. Rollback for reversible actions.
3. Backup target and source setup.
4. Migration dry run before real migration.

Beginner tip:

Do a dry run first when moving large sets of data.

### Goal: "I want my shortcuts to feel personal"

Use:

1. Shortcut catalog create and list.
2. Shortcut categories.
3. Shortcut presets.
4. Shortcut dashboard.

Beginner tip:

Create one preset named "Morning" with your top three locations.

### Goal: "I want confidence that my setup is healthy"

Use:

1. Integration health check.
2. Hotkey diagnostics.
3. NVDA readiness checks.

Beginner tip:

Run health checks after major updates.

## Calm Recovery When Something Goes Wrong

If something feels off, use this order:

1. Stop and press Grave+Escape.
2. Open hotkeys help with Grave+Slash.
3. Open fallback options.
4. Check operation journal.
5. Roll back the last reversible action.

You are not failing when this happens.
This is normal learning behavior.

## Gentle Learning Plan (7 Days)

1. Open palette.

3. Ask for available hotkeys.

### Day 2

2. Read slot description.

2. Add two notes.

### Day 4

1. Create one task.
2. Complete one task.
3. Add one diary entry.

### Day 5

1. Run one retrieval query.
2. Revisit one prior result.

### Day 6

1. Create one shortcut.

3. Add it to a preset.

### Day 7

1. Run backup setup.
2. Check integration health.

## Friendly Rules for Confidence

1. Keep names short and obvious.
2. Use categories only when needed.
3. Prefer one clear action over many quick actions.
4. Use rollback and backups as normal tools, not emergency tools.

## Quick Reference

1. Open palette: NVDA+Shift+P
2. Quick capture: Grave+Q
3. Summarize selection: Grave+S
4. What can I press: Grave+Slash
5. Emergency stop: Grave+Escape

## Final Encouragement

You do not need to master everything at once.
Small repeatable actions create confidence.
If you can open the palette, capture one idea, and complete one task, you are already succeeding.
